# Metrics & Analytics

## Overview

This document outlines the key metrics to be collected from the applications to measure success, user engagement, and system performance. Metrics are categorized by their purpose and collection method.

## User Engagement Metrics

### Event Participation Metrics

- **M1**: Track number of event attendees (actual check-ins/attendance)
- **M2**: Track event sign-ups/registrations
- **M3**: Calculate conversion rate (attendees vs. sign-ups) to identify drop-off or unexpected show-ups
- **M4**: Track event views-to-signup ratio (users who view event details vs. those who register)
- **M5**: Monitor repeat attendance rate (users attending multiple events)

### Event Creation Metrics

- **M6**: Count number of events created by users (non-admin)
<!-- - **M7**: Track average events created per active user -->
- **M8**: Monitor event creation frequency (daily/weekly/monthly trends)
- **M9**: Track event categories/types to identify most popular event formats (entertainment, conference, work)

### Profile Interaction Metrics

- **M10**: Count profile visits (total and unique)
- **M11**: Track profile views per user
- **M12**: Monitor profile completion rate (percentage of profile fields filled)
